# gamefactory Skills 01-05

## Skill 01: Decouple Game Style Learning From Action Control
- evidence: `README.md`
- method: Retain the open-domain prior of a pre-trained large video model while learning action control through a separate lightweight control module.
- transfer: Interactive video systems generalize better when style/world priors and control adaptation are not forced into the same training phase.

## Skill 02: Use Multi-Phase Training to Preserve Open-Domain Generalization
- evidence: `README.md`
- method: Train in phases so the control module can specialize without collapsing the base model's ability to create novel scenes and styles.
- transfer: Generalizable world-model training often needs phased adaptation instead of one end-to-end fine-tune.

## Skill 03: Treat Generative Interactive Video as a New-Game Creation Surface
- evidence: `README.md`
- method: Frame the output not merely as controllable gameplay continuation, but as the creation of new games from interactive video generation.
- transfer: This shifts evaluation toward scene generalization and novelty rather than only imitation of one environment.

## Skill 04: Build Dataset Diversity Through Controlled Environment Factors
- evidence: `README.md`
- method: Vary biome, weather, and time-of-day systematically while collecting long Minecraft clips with randomized actions.
- transfer: Diverse environment factorization is a practical way to improve generalization in game-world video datasets.

## Skill 05: Attach Rich Action Metadata and Scene Descriptions to Every Clip
- evidence: `README.md`
- method: Pair each video clip with action JSON, environment metadata, and a textual description generated by a multimodal model.
- transfer: Action-controllable video datasets become far more reusable when state, control, and scene text are aligned per clip.
